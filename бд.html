База данных

        В нашей работе будет использоваться реляционный тип базы данных,один из самых популярных типов бд,в которых информация хранится в виде таблиц
        Примеры реляционных типов бд в программах:MySQL,PostgreSQL
        Перед тем как начинать работу нужно понимать несколько важных терминов,почему выбран именно реляционный тип БД
        SQL — это структурированный язык запросов, созданный для того, чтобы получать из базы данных необходимую информацию. Если описать схему работы SQL простыми словами, то специалист формирует запрос и направляет его в базу. Та в свою очередь обрабатывает эту информацию, «понимает», что именно нужно специалисту, и отправляет ответ.
        SQL — это не язык программирования, поэтому написать приложение или сайт с его помощью не получится, но при этом внутренняя работа сайта (backend) невозможна без запросов. Поиск информации в Google — это тоже модель использования SQL. Пользователь задает параметры, которые его интересуют, и отправляет запрос на сервер; затем происходит магия и в поисковой выдаче появляются результаты, соответствующие именно этому запросу.
        SQL-запросы обращаются к данным в виде таблиц, то есть к реляционным базам данных. Упрощенный вариант такой базы — это таблицы Excel, в которых информация также упорядочена в столбцы и строки.

       

Основные понятия реляционной модели:
  1. Отношение — это сама таблица, она двумерная и состоит из столбцов и строк.
  3. Атрибут — столбец в таблице, который содержит один конкретный параметр: название, тип, дату, стоимость или другую характеристику.
  4. Домен — это допустимые значения для каждого атрибута. Например, в столбце «Имя» или «Название» значения должны представлять собой набор буквенных символов, но они не могут начинаться с «ь» или «ъ» и не могут быть записаны числами.
  5. Кортеж (строка или запись) — это табличная строка с порядковым номером, в которой содержится информация об одном конкретном объекте.
  6. Значение — элемент таблицы, который находится на пересечении столбцов и строк.
  7. Ключ — это самый важный столбец в таблице, за счет этих значений и происходит взаимодействие в реляционной базе данных, он связывает таблицы между собой.

Исходя из знания и представления работы с реляционной базой данных и sql, для создания своей базы данных, началом послужит следующие цели, а именно:
  1. Цели;
  2. Создание таблиц и внешних ключей;
  3. Связывание таблиц с помощью внешних ключей;
  4. Создание связи по внешнему ключу;
  5. Создание диаграммы сущность-связь.
